# Qwen3-TTS 기사 요약

- 문서 목적: Qwen3-TTS 관련 핵심 내용을 빠르게 공유하고, 프로젝트(Tangshi)에서 바로 적용 가능한 판단 기준을 제공
- 작성일: 2026-02-16
- 작성자: CH (GPT/Codex)
- 기준 글: 파이토치 한국 사용자 모임 게시글(2026-01-25)

---

## 1. 한눈에 요약

Qwen3-TTS는 "실시간성(12Hz)"과 "고음질(25Hz)"을 분리한 오픈소스 TTS 계열입니다. 
핵심 포인트는 다음 4가지입니다.

1. 학습 데이터 규모가 매우 큼: 500만 시간 이상, 10개+ 언어
2. 초저지연 스트리밍 가능: 12Hz 계열에서 첫 패킷 지연 약 97~101ms(모델 크기별)
3. 제어 기능이 강함: 3초 음성복제, 텍스트 기반 Voice Design, 지시문 기반 스타일 제어
4. 공개 범위가 넓음: 모델/토크나이저/기술문서가 Apache 2.0 기반 공개

---

## 2. 모델 라인업 (기사 기준)

### 공통
- 한국어 포함 다국어 지원
- 스트리밍 지원
- 0.6B / 1.7B 크기 제공(일부 변형)

### 주요 변형
1. Base
- 파운데이션 모델
- 3초 기반 빠른 음성복제
- 파인튜닝 베이스로 적합

2. VoiceDesign
- 텍스트 설명으로 목소리 생성
- 예: "중저음 뉴스 앵커 톤" 같은 자연어 지시

3. CustomVoice
- 프리셋 음색 + 지시 기반 정밀 제어

---

## 3. 기술 핵심 정리

### A) 12Hz 토크나이저 (실시간 우선)
- 12.5Hz, 다중 코드북(RVQ 기반)
- 의미/음향 정보를 계층적으로 분리
- 인과(Causal) 설계로 look-ahead 없이 스트리밍
- MTP(Multi-Token Prediction)로 잔여 코드북을 빠르게 예측
- 결과: 첫 오디오 패킷 지연이 매우 짧음

### B) 25Hz 토크나이저 (음질 우선)
- 25Hz, 단일 코드북
- Flow Matching + DiT 기반 재구성
- 청크 처리로 스트리밍 가능하지만 12Hz보다 지연은 큼
- 용도: 더빙/오디오북 등 음질 우선 시나리오

### C) 아키텍처 포인트
- Qwen3 LM 백본 + 화자 인코더 공동학습
- Dual-track 입력(텍스트/음성 토큰 동시 처리)
- ChatML 포맷 지시문 처리에 최적화

---

## 4. 학습/성능 포인트 (기사에서 강조한 내용)

1. 학습 단계
- General(대규모 다국어) -> High-Quality(품질 정제 CPT) -> Long-Context(최대 32,768 토큰)
- Post-training에서 DPO 기반 선호 최적화

2. 기능 성능
- 3초 제로샷 음성복제
- 장문(10분+) 생성 안정성 강화
- 교차언어 성능 개선(기사 내 Zh->Ko WER 개선 사례 제시)

3. 벤치마크 언급
- 다국어, Zero-shot, 지시이행 벤치에서 강한 결과를 제시
- 다만 실제 서비스 품질은 환경/프롬프트/레퍼런스 오디오 품질에 따라 달라짐

---

## 5. 설치/실행 최소 절차 (기사 예시 요약)

1. Python 3.12 가상환경
2. `pip install -U qwen-tts`
3. 선택: `flash-attn` 설치(메모리/속도 보조)
4. `Qwen3TTSModel.from_pretrained(...)`로 모델 로드
5. `generate_voice_design(...)` 등 함수로 WAV 생성

주의:
- GPU/드라이버/CUDA 환경에 따라 초기 세팅 이슈 가능
- 대형 모델(1.7B)은 VRAM 여유 필요

## 6. Tangshi 프로젝트 적용 관점

### 즉시 적용에 적합한 전략
1. 1차 PoC는 12Hz-0.6B로 시작
- 이유: 지연/자원 부담 균형이 좋고 반복 테스트가 빠름

2. 품질 기준 통과 시 1.7B 비교
- 같은 문장/같은 화자샘플로 A/B 청취

3. 배치 생성 파이프라인 분리
- 개발: 샘플 실험 스크립트
- 운영: `scripts/generate_tts.py` 배치 전용(재시도/로그/중단복구)

### 추천 검증 항목
- 발음 정확도(중국어 고유명사/고전 어휘)
- 운율 자연스러움(낭송 톤)
- 문장 길이 증가 시 안정성
- 생성 속도(초당 처리량, 첫 응답 지연)
- 파일명/메타 규칙(`public/audio/` 기존 자산과 충돌 여부)

## 7. 의사결정 체크리스트

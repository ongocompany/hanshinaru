## 2. 출생지(birthplace) 필드 형식

```json
"birthplace": {
  "name": "한국어 지명",
  "nameZh": "중국어 지명 (현재 행정구역)",
  "lat": 위도(소수점 2자리),
  "lng": 경도(소수점 2자리)
}
```

### 작성 예시 (이백)
```json
"birthplace": {
  "name": "사천성 면양 (출생지: 쇄엽성)",
  "nameZh": "四川省 綿陽縣 (出生: 碎葉城)",
  "lat": 31.47,
  "lng": 104.73
}
```

### 규칙
- `name`: 한국어 한자음 표기 (예: "하남성 공현")
- `nameZh`: 중국어 원문 + 괄호 안에 현재 지명 (예: "河南省 鞏縣(지금의 鞏義市)")
- `lat`, `lng`: Google Maps에서 해당 지명 검색 후 소수점 2자리까지
- 출생지가 불명확한 경우: 주요 활동지를 대신 사용하고 name에 "(활동지)" 표기
- 완전히 알 수 없는 경우: birthplace 필드 자체를 생략

## 3. 교유관계(relations) 필드 형식

```json
"relations": [
  {
    "targetId": "상대 시인 titleId 또는 외부인물 고유ID",
    "type": "관계유형(영어)",
    "label": "화면표시용(한국어)",
    "desc": "관계 설명 1줄"
  }
]
```

### 작성 예시 (이백) — 프로젝트 내 시인 관계
```json
"relations": [
  {
    "targetId": "C341",
    "type": "friend",
    "label": "친구(벗)",
    "desc": "두보와 이백은 744년 낙양에서 만나 의기투합"
  },
  {
    "targetId": "C347",
    "type": "respect",
    "label": "존경",
    "desc": "〈贈孟浩然〉에서 '吾愛孟夫子 風流天下聞'이라 읊음"
  }
]
```

### 작성 예시 — 외부 인물 관계 (시인이 아닌 인물)
시인이 아니더라도 역사적으로 의미 있는 인물과의 관계는 기록 가능.
외부 인물은 `targetId`에 `EXT_` 접두사를 붙이고, `targetName`/`targetNameKo` 필드를 추가.

```json
{
  "targetId": "EXT_xuanzong",
  "type": "patron",
  "label": "총애받음",
  "targetName": "唐玄宗",
  "targetNameKo": "당 현종",
  "desc": "현종이 이백을 한림학사로 임명, 이후 금방환향(賜金放還)"
}
```

| 필드 | 설명 | 필수 |
|------|------|------|
| `targetId` | `EXT_` + 영문식별자 (예: `EXT_xuanzong`, `EXT_geshu_han`) | O |
| `targetName` | 중국어 이름 | O |
| `targetNameKo` | 한국어 이름 | O |
| 나머지 | type, label, desc는 기존과 동일 | O |

**외부 인물 targetId 네이밍**: `EXT_` + 인물 병음(소문자, 밑줄 구분)
- 예: 唐玄宗 → `EXT_xuanzong`, 哥舒翰 → `EXT_geshu_han`, 楊貴妃 → `EXT_yang_guifei`

### type 종류

| type | 의미 | 화면 화살표 | 사용 예 |
|------|------|------------|---------|
| `friend` | 친구/벗 | 양방향 ←→ | 이백 ↔ 두보 |
| `respect` | 존경함 | 단방향 → | 이백 → 맹호연 |
| `admirer` | 존경받음 | 단방향 → | 맹호연 → 이백 |
| `patron` | 추천받음/후원받음 | 단방향 → | 이백 → 하지장 |
| `recommend` | 추천함/후원함 | 단방향 → | 하지장 → 이백 |
| `teacher` | 스승 | 단방향 → | 가도 → 한유 (한유가 스승) |
| `student` | 제자 | 단방향 → | 한유 → 가도 (가도가 제자) |
| `rival` | 라이벌/병칭 | 양방향 ←→ | 원진 ↔ 백거이 |
| `colleague` | 동료/동시대 병칭 | 양방향 ←→ | 왕창령 ↔ 이백 |

### 규칙
- **양쪽 다 작성**: A→B 관계를 넣으면 B→A도 반드시 넣을 것
  - 예: 이백에 `friend → C341(두보)` 넣으면, 두보에도 `friend → C375(이백)` 넣기
- `desc`는 가능하면 구체적 연도/작품명 포함 (1줄 이내)
- 교유관계가 확인되지 않는 시인: `"relations": []` (빈 배열)
- 프로젝트 내 시인(C328~C403) 간 관계 + 역사적으로 의미 있는 외부 인물 관계도 기록 가능
- 외부 인물 관계 시 반드시 `targetName`, `targetNameKo` 필드 포함

## 6. 참고사항

- 각 시인의 전기 정보는 `db_author.with_ko.json`의 `bioKo` 필드에 이미 있으니 참고할 것
- 기존 완료 4명(C341, C347, C375, C398)은 조사 불필요
- 출생지 좌표는 Google Maps에서 지명 검색 후 위도/경도 복사
- 생몰년 미상/정보 부족 시인은 가능한 범위까지만 조사하고 불명은 생략 가능

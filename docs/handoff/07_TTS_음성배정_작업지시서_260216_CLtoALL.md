# TTS 음성 배정 작업 지시서

- 작성일: 2026-02-16
- 작성자: CL (Claude)
- 수신: ALL (GPT/Gemini/기타 AI)
- 목적: 320수 각각에 최적 화자 + 낭송 스타일을 배정하는 매핑 파일 생성

---

## 1. 배경

Qwen3-TTS로 당시삼백수 320수의 낭송 오디오를 생성할 예정.
TTS 품질이 높아 시마다 어울리는 화자/스타일을 다르게 배정해야 함.
사람이 320수를 일일이 판단하기엔 비효율적이므로, **AI가 시 내용을 분석하여 자동 배정**하고 사람은 검수만 하는 구조.

## 3. 사용 가능한 낭송 스타일 (instruct 프리셋)

| ID | 한국어 | 중국어 instruct 텍스트 |
|----|--------|----------------------|
| majestic | 웅장 낭송 | 庄重缓慢地朗诵，低沉浑厚的声音，带有古典诗歌的韵律感，在逗号和句号处自然停顿 |
| calm | 담백 낭송 | 平和淡然地朗诵，语速适中，声音清晰自然，不带过多感情色彩 |
| sad | 슬픈 감성 | 深沉悲伤地朗诵，声音低沉缓慢，带有叹息和怀念之感 |
| bright | 경쾌한 | 轻快明亮地朗诵，声音清脆，语速稍快，充满活力和喜悦 |
| slow | 매우 느린 | 极其缓慢地朗诵，每个字都清晰饱满，字与字之间有明显停顿，适合初学者跟读 |

**참고**: instruct는 자유 텍스트이므로 프리셋에 없는 스타일도 중국어로 직접 작성 가능.
예: "豪放不羁地朗诵，声音高亢激昂，如李白饮酒后的狂放" (호방한 낭송)

## 4. 작업 내용

### 4-1. 입력 데이터
- 파일: `public/index/poems.full.json`
- 320수, 각 시에 다음 정보가 있음:
  - `poemNoStr`: 번호 (001~320)
  - `title.zh`: 제목 (한자)
  - `poet.zh`: 시인
  - `poemZh` 또는 `poemSimp`: 시 본문
  - `translationKo`: 한국어 번역
  - `category`: 시체 (五言古詩, 七言律詩 등 8종)
  - `meter`: 5 또는 7 (오언/칠언)
  - `jipyeongZh`: 집평 (비평 텍스트, 있는 경우)

### 4-2. 출력 파일
- 경로: `public/index/tts_voice_map.json`
- 형식:

```json
[
  {
    "poemNoStr": "001",
    "titleZh": "感遇",
    "poetZh": "張九齡",
    "speaker": "Uncle_Fu",
    "instruct": "calm",
    "instructCustom": "",
    "reason": "은유적 감회시, 담담하고 깊은 톤이 적합",
    "confidence": "high"
  },
  {
    "poemNoStr": "015",
    "titleZh": "望嶽",
    "poetZh": "杜甫",
    "speaker": "Dylan",
    "instruct": "majestic",
    "instructCustom": "",
    "reason": "태산의 웅장함을 노래, 청년의 패기",
    "confidence": "high"
  },
  {
    "poemNoStr": "029",
    "titleZh": "玉臺體",
    "poetZh": "權德輿",
    "speaker": "Serena",
    "instruct": "calm",
    "instructCustom": "温柔细腻地朗诵，声音轻柔，带有闺阁的婉约之感",
    "reason": "옥대체(규방시), 여성 화자가 자연스러움",
    "confidence": "high"
  }
]
```

### 4-3. 필드 설명

| 필드 | 설명 |
|------|------|
| speaker | 화자 ID (Uncle_Fu, Dylan, Eric, Vivian, Serena 중 1) |
| instruct | 프리셋 ID (majestic, calm, sad, bright, slow 중 1) |
| instructCustom | 프리셋으로 부족할 때 직접 작성하는 중국어 instruct (빈 문자열이면 프리셋 사용) |
| reason | 배정 이유 (한국어, 1줄) — 형이 검수할 때 판단 근거 |
| confidence | 확신도: "high" / "medium" / "low" — low인 것만 형이 집중 검수 |

## 5. 배정 가이드라인

### 화자 선택 기준

1. **Uncle_Fu** (기본값으로 가장 많이 배정될 것)
   - 장중한 시, 역사시, 회고시, 노년의 감회
   - 두보의 대부분, 왕유의 산수시, 위응물 등

2. **Dylan**
   - 청년의 기상, 맑고 자연스러운 낭송
   - 이백의 일부, 자연/여행시, 밝은 톤의 시

3. **Eric**
   - 호방하고 활력 있는 시
   - 이백의 호방한 시, 변새시(전쟁시), 기개 있는 시

4. **Vivian**
   - 밝고 예리한 여성 목소리
   - 봄/꽃 시, 밝은 경물시, 짧고 경쾌한 절구

5. **Serena**
   - 부드럽고 따뜻한 여성 목소리
   - 규원시(閨怨), 궁원시(宮怨), 여성 화자 시, 이별의 슬픔

### 스타일 선택 기준

- **majestic**: 산하의 장관, 역사적 감개, 국가적 비장함
- **calm**: 일상의 감회, 자연 관조, 담백한 서정
- **sad**: 이별, 그리움, 유배, 전쟁의 참상, 늙음의 탄식
- **bright**: 봄날, 기쁜 소식, 자연의 아름다움, 경쾌한 절구
- **slow**: 사용하지 않음 (배치 생성에서는 제외, 테스트용)

### 특수 케이스

- **같은 시인이라도 시마다 다를 수 있음**: 이백은 호방할 때도, 슬플 때도 있음
- **화자 성별 판단**: 시의 내용에서 화자가 여성인 경우 (규방에서 남편을 기다리는 등) 여성 화자 사용
- **확신 없으면 "low"**: 어떤 화자/스타일이 맞을지 모호하면 confidence: "low"로 표시

## 7. 검증 방법

작업 완료 후 다음을 체크:
1. 320수 전부 매핑되었는지 (누락 없는지)
2. speaker 값이 5명 중 하나인지
3. instruct 값이 프리셋 ID 중 하나인지 (또는 instructCustom에 중국어 텍스트)
4. confidence: "low"인 항목이 몇 개인지 (형이 검수할 분량)
5. JSON 문법 오류 없는지

## 8. 작업 후 다음 단계

1. 형이 매핑 파일 검수 (특히 confidence: "low" 항목)

3. 확정 후 배치 생성 스크립트 실행 (320수 × WAV → MP3 변환)
4. `public/audio/` 폴더에 최종 MP3 배치

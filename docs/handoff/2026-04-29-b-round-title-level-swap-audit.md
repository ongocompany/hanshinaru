# 핵심 판단과 이유

## 1. B 우선 진행 (D 다음 세션)
**판단**: 형 결정 — "검증이 더 중요. 효과 0일 수도 있지만 만의 하나면 사이트 신뢰도 전체 하락".
**이유**: 결과적으로 발견 41수 + 핵심 김창흡 잔여 인식 정정. 검증 우선 결정의 가치 입증.

## 2. 시제 단위 cross-check 도구 신규 작성
**판단**: `audit_index_vs_db.mjs`(분량 비교, 어제 사용)에 추가로 `audit_title_swaps.mjs`(시제 비교) 작성.
**이유**: 분량 비교는 fuzzy 매칭 결함의 큰 케이스만 잡음. 시제 1:1 매칭이 옅은 swap을 잡을 수 있음.

## 3. 어제 정정 11건 화이트리스트로 false positive 제거
**판단**: KNOWN_FIXES 배열에 어제 정정된 매핑 (이언진→이숭인 등) 11개 등록. 색인이 잘못된 시인을 지명해도 DB가 정답이면 정상 처리.
**이유**: 어제 정정 후 색인은 fuzzy 결함 raw, DB가 정답. 화이트리스트 없으면 어제 정정한 499수가 전부 의심으로 잡힘.

## 4. 4 Sonnet subagent 병렬 web 검증 (Opus 아님)
**판단**: 어제 9 swap web 검증과 동일 패턴.
**이유**: 검증 작업은 자료 수집·판단 위주. Sonnet이 wiki + WebSearch tool 더 효율적.

## 5. 김창흡 잔여 52수 → 18수 정확화
**판단**: 어제 핸드오프 "잔여 52수는 진짜 김창흡"이 분량 단위 결과였음. 시제 단위로 추가 34수 분리 후 잔여 18수.
**이유**: 어제는 색인 vs DB **분량** 비교만 했지만 같은 시인이라도 시제 단위로 각자 다른 swap 가능성이 있음.

- **자동 색인 추가 swap 가능성** — B 라운드는 DB 시인 ≠ 색인 시인 케이스. **DB에는 있는데 색인에 없는 시제** (1577수)나 **색인에 다중 시인 후보** 케이스는 본 라운드에서 안 잡힘. 별 라운드 가능.
- **김창흡 18수 진짜 시 검증 권장** — 잔여 18수가 정말 김창흡(三淵集 시) 맞는지 sample 검수 가능.

## 작업 흐름

- **시제 단위 검사가 분량 단위 검사보다 강력** — 본 라운드 입증. 향후 fuzzy 결함 의심 케이스는 시제 단위 우선.
- **subagent web 검증의 가치** — 결정적 증거(이준 사망 연도·천관사 파한집·御溝 자구 일치) 모두 web으로 확보. 색인+분량+web 3중 검증 필수 정책 확립.
- **production DB 직접 쓰기 신중 절차 유지** — 본 라운드 9 INSERT + 41 row UPDATE + 7 bio 갱신. 매 단계 dry-run + 검증 + 형 OK.

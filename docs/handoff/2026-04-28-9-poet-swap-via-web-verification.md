# 핵심 판단과 이유

## 1. C 옵션 (web 전수 검증) 채택
**판단**: 색인 cross-check만으로 confident하지 않고, 전수 web 검증.
**이유**: 진우형 결정. 자동 매칭이 안 됐던 허목 case는 풍격만으로 김시습 추정했으나, web 검증 결과 사실은 허봉(許篈)임이 색인 100% 일치로 확인. **풍격 기반 추측은 잘못 갈 수 있음**. 색인+분량+web 3중 검증이 안전.

## 2. Sonnet subagent 활용 (Opus 아님)
**판단**: 5 web 검증 subagent를 Sonnet으로.
**이유**: 검증 작업은 자료 수집 + 판단 위주. Opus의 학술 깊이가 절대 필요하지 않음. Sonnet이 web tool 더 효율적. commentary 재생성은 Opus 유지 (학술 격조).

## 3. 시인별 chunk 분할 (도은집·근재집과 다른 방식)
**판단**: 5 chunks를 시인별로 묶음 (강정일당 24, 신숙주 24, 이행 24, 김창협 22, 허봉+4시인 26).
**이유**: 한 chunk에 한 시인이면 subagent의 컨텍스트 집중 → commentary 품질·일관성 ↑. 도은집·근재집은 한 시인 시집이라 분량으로만 나눴지만, 이번엔 9 시인 혼합이라 시인 단위가 자연스러움.

## 4. 잘못 매핑 시인 row 보존 + bio "혼동" 메모
**판단**: poet#3412 등 9명 row 삭제 X. life/era/bio 정확화 + "이전 매핑은 ... 한자 성씨 혼동 결과" 메모.
**이유**: 진짜 시인들. 향후 진짜 시문 적재 시 매핑 대상. 어제 도은집/근재집과 동일 정책.

## 5. 이현보·김창흡 분리 (전체 swap 아님)
**판단**: 이현보 44수 중 24만 이행으로, 나머지 20은 진짜 이현보 유지. 김창흡 74수 중 22만 김창협으로, 나머지 52는 진짜 김창흡 유지.
**이유**: web 검증 결과 두 시인 모두 진짜 시 + 다른 시인 시가 섞여있음. id 그룹이 깔끔히 분리됨 (이현보 잔여 48035~48054, swap 48257~48280; 김창협 swap 48055~48076).

## 6. 허목 → 김시습 가설 기각, 허봉으로 정정
**판단**: 풍격 기반 김시습 추정을 web 검증으로 기각, 색인 100% 일치 허봉으로 결정.
**이유**: 매월당집 「贈峻上人二十首」는 「其一~其二十」 연번 형식인 반면 DB 20수는 각기 다른 시제. 색인 「荷谷先生集 詩鈔」 20건과 100% 일치. 풍격 기반 추측의 위험성 보여줌.

## 7. 한자 성씨 fuzzy 매칭 결함 패턴 명시
**판단**: 9 swap 모두 "같은 한자 성씨 / 시각적 유사" 공통점 명시 → 색인 파이프라인 결함으로 진단.
**이유**: 다음 라운드에서 자동 색인기 검토·재실행 시 결함 수정 우선순위. 그리고 비슷한 swap 더 발견 가능성 알림.

## 1) 지훈 담당 범위 요약

1. TTS(Qwen3-TTS) 데모 검증 및 대량 생성 파이프라인 준비
2. 한시 작성 도우미 Phase 1(엔진 코어) 구현
3. 한시 작성 도우미 Phase 4(AI 연동) 설계/구현

---

## 2) 우선순위

### P0 (즉시)
- TTS 샘플 5~10수 생성 및 음성 품질 검증
- 한시 도우미 설계문서(06~09) 재정독 후 엔진 스펙 확정

### P1 (핵심 구현)
- 평수운/평측 데이터 파이프라인 정비
- 율격 템플릿 + 검사기(형식/운/평측/병) 구현
- `public/index/poems.full.json` 기준 샘플 10수 자동 검증

### P2 (고도화)
- AI 연동(운자 추천, 구제 후보 추천, 비교 리포트)
- 민철 UI(Phase 2~3)와 엔진 입출력 인터페이스 고정

---

## 3) 작업 항목 상세

## A. TTS 작업

### 목표
- Qwen3-TTS 기반으로 당시는 자연스러운 발화 품질 확보
- 이후 313수 일괄 생성 가능한 상태로 스크립트 정비

### 대상 파일
- `scripts/generate_tts.py`
- `public/index/poems.full.json`
- `public/audio/`
- `test_tts.html`

### 실행 계획
1. `poems.full.json`에서 대표 시 5~10수 선정
2. 샘플 음성 생성(중국어 발화 우선)
3. 발음/속도/톤 기준으로 프리셋 후보 정리
4. 형님 피드백 반영 후 일괄 생성 파라미터 확정
5. `generate_tts.py`를 313수 배치 처리 중심으로 보강

### 완료 기준
- 샘플 청취 후 사용할 기본 보이스/속도 확정
- 배치 생성 스크립트가 실패 재시도/로그를 포함해 실행 가능
- 오디오 네이밍 규칙이 기존 `public/audio/` 자산과 충돌 없음

---

## B. 한시 작성 도우미 Phase 1 (엔진 코어)

### 목표
- 평수운 기반 규칙 검사 엔진의 최소 실사용 버전 완성

### 필독 문서
- `docs/FromJin/06_시작성도우미제작계획to지훈.md`
- `docs/FromJin/07_시작성도우미_전체프레임워크설계_260216_Codex.md`
- `docs/FromJin/08_시작성도우미_로직설계_엔진_AI_260216_Codex.md`
- `docs/FromJin/09_시작성도우미_UI기획_260216_Codex.md`

### 대상 파일
- `public/index/pingshui_yun.json`
- `scripts/build_pingshui_db.js`
- `scripts/generate_pinyin_pingze.py`
- (신규 예정) `public/index/*.json` 엔진 데이터
- (신규 예정) `tools/` 또는 `scripts/` 내 검사기 모듈

### 실행 계획
1. 평수운 DB 구조 점검 및 누락 필드 보강 항목 정의
2. 병음/평측 생성 스크립트 확장(재생성 가능 구조)
3. 율격 템플릿 정의
   - 오언/칠언
   - 절구/율시
   - 평기/측기
4. 검사기 구현
   - 형식 검사(행수/자수)
   - 운 검사(압운 일치)
   - 평측 검사(규칙 매칭)
   - 병 탐지(기본 수준)
5. `poems.full.json` 10수 샘플 자동 테스트

### 완료 기준
- 입력 텍스트에 대해 검사 리포트(JSON) 산출 가능
- 샘플 10수 기준 정상/오류 유형이 구분되어 출력
- 민철 UI에서 바로 붙일 수 있는 결과 포맷 정의 완료

## C. 한시 작성 도우미 Phase 4 (AI 연동)

### 목표
- 규칙 검사 결과를 기반으로 실질적인 수정 제안 제공

### 범위
- 운자 추천
- 구제(拗救) 후보 제안(기본 수준)
- 수정 전/후 비교 리포트

### 실행 계획
1. 엔진 출력 스키마 고정(오류 코드, 위치, 근거)
2. AI 프롬프트 템플릿 설계(규칙 기반 + 제안 분리)
3. 추천 결과를 구조화된 JSON으로 반환하도록 설계
4. UI 연결용 응답 포맷(민철 담당 Phase 2~3) 동기화

### 완료 기준
- 동일 입력에 대해 재현 가능한 추천 결과 생성
- 제안 근거(운/평측 규칙) 포함
- UI에서 카드형으로 표시 가능한 응답 구조 확보

---

## 4) 협업 인터페이스(민철 연동)

1. 엔진 입력
- 시 형식(오언/칠언, 절구/율시, 평기/측기)
- 본문 원문(행 단위)
- 선택 운목(선택 입력)

2. 엔진 출력
- `summary`: 총평
- `lineChecks[]`: 행별 형식/평측/압운 결과
- `issues[]`: 오류 코드, 위치, 설명, 심각도
- `suggestions[]`: 수정 제안, 대체 글자/구절, 근거

3. 전달 방식
- 초기: 로컬 JSON/함수 호출
- 후속: 필요시 API 엔드포인트화

# 근체시 작성 도우미 전체 프레임워크 설계안 (v1)

기준 문서:
- `docs/FromJin/06_시작성도우미제작계획to지훈.md`

목표:
- 근체시 학습 + 창작을 한 페이지/한 플로우에서 연결
- 규칙 판정 신뢰성(엔진)과 창작 보조 유연성(AI)을 분리
- 초기에는 프론트 중심으로 시작하고, AI/저장/공유는 단계적으로 확장

---

## 1. 제품 구조(상위 레벨)

### 1) 학습 영역
- 시체별 규칙 설명(절구/율시/배율, 평측, 압운, 대우)
- 대표 작품 하이라이트(압운/대우/구조 표시)

### 2) 창작 영역
- 7단계 작성 도우미:
  1. 형식 선택
  2. 주제/소재 입력
  3. 구상(한글 스토리보드)
  4. 운목/운자 선정
  5. 압운 자리 채우기
  6. 본문 작성(모드 분기)
  7. 감상 + 리포트

### 3) 결과 영역
- 형식 리포트(엔진 결과)
- 의미/자연스러움/대우 코칭(AI 결과)
- 저장/내보내기/공유

---

## 2. 아키텍처 원칙

### 원칙 A: 판정은 엔진, 코칭은 AI
- 엔진: 자수/구수/평측/압운/대우 좌표/요구 판정
- AI: 후보 추천, 자연스러운 해설, 검증용 번역

### 원칙 B: 동일 I/O 계약
- 모든 단계 결과를 JSON 상태로 누적
- UI는 상태 기반 렌더링(재현 가능)

### 원칙 C: 단계형 확장
- v1: 로컬 저장 + 규칙 검사 + 기본 AI 보조
- v2: 계정/클라우드 저장 + 커뮤니티
- v3: 고급 요구/대우 강화 + 학습 분석

---

## 3. 시스템 구성

### 3.1 프론트엔드
- 추천: Vanilla JS 또는 React 중 택1
- 상태 모듈:
  - `formState`(형식/주제/구상/본문)
  - `ruleState`(검사 결과/경고)
  - `assistState`(AI 추천/번역/코멘트)
  - `storageState`(저장/버전)

### 3.2 룰 엔진(독립 모듈)
- 입력: 작품 상태(JSON)
- 출력: 판정 결과(JSON)
  - `errors[]`, `warnings[]`, `suggestSlots[]`, `rhymeCheck`, `toneCheck`, `coupletCheck`

### 3.3 AI 어시스트 레이어
- 입력: 엔진 결과 + 사용자 조건
- 출력: 후보/설명 중심 JSON
- 엔진 제약을 먼저 통과한 슬롯에만 AI 후보 허용

### 3.4 데이터 레이어
- 정적 DB:
  - 운서 매핑(`char -> 운부`, `운부 -> chars`)
  - 평측 매핑(`char -> 평/측`)
  - 형식 템플릿(`오언/칠언`, `절구/율시/배율`, `평기/측기`)
- 사용자 데이터:
  - v1: LocalStorage/IndexedDB
  - v2: Supabase/Firebase

---

## 4. 실행 플로우(요약)

1. 형식 선택 → 템플릿 로딩
2. 주제/소재/구상 입력 → 구조 검사(빈칸/구수)
3. 운목/운자 선택 → 압운 자리 잠금
4. 본문 작성(자유/실시간)
5. 엔진 검사 실행(학인)
6. 필요 시 AI 코칭 호출
7. 저장/내보내기/공유

---

## 5. 모드 전략

### 모드 A: 자유 작성 + 일괄 검사
- 장점: 창작 자유, 숙련자 친화
- 주 사용 기능: 최종 학인 리포트, AI 후처리 코칭

### 모드 B: 실시간 신호등 가이드
- 장점: 초보자 진입 장벽 낮춤
- 주 사용 기능: 슬롯별 경고, 운서 기반 후보, 제한적 AI 보조

---

## 6. 품질/운영 원칙

1. 엔진 메시지는 반드시 좌표+근거 포함
2. AI 출력은 자동 적용 금지(사용자 선택)
3. 긴 시(배율/장편)는 련 단위 계산/캐시 적용
4. 동일 입력 AI 결과 캐시로 비용 절감

---

## 7. 단계별 로드맵

### Phase 1 (MVP)
- 7단계 UI 뼈대
- 엔진 기본 검사(자수/구수/압운/기초 평측)
- 로컬 저장/JSON 내보내기

### Phase 2
- 요체(拗救) 처리
- 대우 구조 힌트
- AI 검증 번역/구상 일치도 코멘트

### Phase 3
- 고급 병 탐지 확장
- 공유 링크/시화 PNG
- BaaS 기반 계정/클라우드 저장(선택)

---

## 8. 내가 추가 제안하는 포인트

1. **검사 강도 프리셋**
- 초급/중급/고급으로 검사 규칙 레벨을 분리하면 좌절감 감소

2. **학습 리플레이**
- 한 작품을 완성하기까지의 수정 히스토리를 타임라인으로 저장

3. **오류 우선순위 정렬**
- 첫 화면은 “치명 오류 3개”만 먼저 보여주고, 상세는 접기

4. **템플릿 예열 버튼**
- 형식 선택 직후 “샘플 골격 자동 채우기” 제공하면 공백 공포 줄어듦

고생했네. 훨씬 보기 좋음. 몇가지 수정사항.

1. 시 모달에서 아코디언 없애자는건 시 한편  볼 경우를 말하는 것이었음.  작가모달에서 하단 작품리스트마저 아코디언이 없어졌음 →작품 리스트는 기존 아코디언 방식 복구
2. 기존 로직 ( 매 시인당 맨 위에 있는 첫 시는 이름 노출 시키는것 )과 새로운 로직 (모든 작품에 제목과 시인 노출)이 충돌해서 시인마다 첫 시에는 이름이 두번 등장 → 기존로직(시인당 첫시 이름노출) 삭제할것 (아래 4-C참조)
3. 주석 기능 : 집평(한자로된 원문)에 주석이 있는 경우가 있는데 기능이 작동 안하고 있으니 확인 바람
    1. 본문에 주석 밑에 줄 생기는거 안보이게 처리
    2. 본문에 주석 번호 (시인 이름이나, 시 제목, 집평 모두) 안보이게처리 (어차피 텍스트 색이 달라서 헷갈릴일 없고 주석 번호는 책이면 필요한데 디지털 상에선 필요가 없음, 색인이 필요 없으므로)
4. 작품 모달 수정
    1. 그림기능 삭제 : 포기. 그림과 함께 시를 보여주는 다른 방법 생각해냄 (하단에 정리)
    2. 시 본문과 번역문을 한 라인에 통합 
    3. 시부분 박스 형태
        1. 시 제목 (한자) - 1 칼럼
        2. 시 제목 (한글) - 1 칼럼
        3. 시인 이름 (한자, 칼럼내에서 오른쪽 정렬) - 1 칼럼 
        4. 시 본문 : 두 컬럼으로 나눠서 왼쪽 한자 , 오른쪽  한글 
        5. 상단 시 박스 상화 좌우 폭은 시에 따라 유동적이게 
        6. 아래로 집평 등등
    4. 이렇게 하면 시 본문 오른쪽이 허전해 보이는 문제 해결됨, 당시 삼백수 책이 딱 이렇게 되어 있음
    
5. 그럼 시 본문과 그림은 어떻게 결합하느냐. 
    1. 이건 전체 시를 다 할수가 없다는걸 깨달았음. 그림이 어울리는 시가 있는 반면 도저히 안되는 시도 있음
    2. 하지만 그림과 시를 꼭 보여주고 싶은 시들의 경우는 메인페이지 히어로 섹션 하단, 타임라인 섹션 위에 오늘의 시라는 새로운 섹션 삽입 (크지 않게 작게 세 편정도 썸네일로 제목과 이미지만 노출)
    3. 클릭하면 하단 전체가 내려가며 딱 시 + 배경그림만 있는 시 화면 노출 
        1. 여기에 보여줄 시는 관리툴에서 만드는게 아니라, 내가 틈틈히 직접 만들어서 이미지로 만들 예정 (이 경우 모바일에서 글자가 너무 작아질거 같긴한데, 모바일에서는 해당 섹션 히든으로 처리해 버리거나, 모바일용 오늘의 시는 제목만 텍스트로 보여주고 본문 링크 걸어주는 방식으로 처리)
        2. 그러니까 관리툴에서는 오늘의 시 리스트만 업데이트 하고 본문하고 태깅만 가능하게 하면 됨 (초반에 한 10개 만들어 둘건데, 랜덤으로 3개만 썸네일처럼 작게 한줄에 보여주고 점차 업데이트 할 예정)
        
6. 관리툴 수정사항
    1. 사진 기능 빠졌으므로 3단으로 갈 필요가 없어졌음. 2단으로 복귀
    2. 왼쪽 시 관리 툴 No 에 해당하는 컬럼이 너무 좁아져서 번호가 0밖에 안보임, 이미지 관리 부분이 빠졌으니, 좌우폭 확인해서 왼쪽 NO 정상적으로 나오도록 조정
    3. 변경 사항이 없을 경우에도 맨 위에 저장버튼 누르면 그냥 json저장하게 변경 (이건 그냥 내가 심리적으로 불안해서 자주 저장하기 때문에…)

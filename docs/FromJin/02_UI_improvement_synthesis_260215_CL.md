## 1. 모든 제안의 공통 진단

세 문서 모두 같은 결론에 도달했다:

> **"감상(예술)"과 "학습(정보)"이 한 공간에서 충돌하고 있다.**

- 시 + 미드저니 그림 = 여백, 분위기, 감성 (세로 흐름)
- 집평 + 주석 + 번역 = 빽빽한 정보, 가독성 (가로 흐름)
- 이 둘이 하나의 수직 스크롤에 섞여서 리듬이 깨진다

현재 코드 근거:
- 모달 폭 `min(1400px, 96vw)` → 너무 넓어서 세로형 시가 허전하게 떠 있음
- `.poem-text-overlay`의 `max-height: 85%; overflow: auto` → 긴 시는 작은 스크롤 박스가 되어 감상 불가
- 본문/집평/주석/번역/심화가 전부 단일 수직 나열 → 끝이 안 보임

## 3. 민철(Claude)의 종합 제안: "한 화면 감상 → 아래로 학습"

모드를 2개로 나누는 대신, **하나의 자연스러운 흐름**으로 해결하자.

### 핵심 컨셉: 두루마리(Scroll) 방식

동양화 감상의 본질은 **두루마리를 펼쳐보는 것**이다.
위에서 아래로, 또는 오른쪽에서 왼쪽으로 천천히 펼쳐가며 보는 경험.
이걸 웹에서 구현하면:

```
┌──────────────────────────────────────┐
│                                      │
│          (미드저니 그림)              │  ← 그라데이션으로
│          배경 이미지 영역             │     아래쪽이 서서히
│                                      │     종이색으로 변함
│     〈春望〉                         │
│                    杜甫               │  ← 시 텍스트는
│                                      │     그림 위에 자연스럽게
│     國破山河在  城春草木深             │
│     感時花濺淚  恨別鳥驚心             │
│     烽火連三月  家書抵萬金             │
│     白頭搔更短  渾欲不勝簪             │
│                                      │
├──── 여기서부터 종이 배경 ─────────────┤  ← 자연스러운 전환
│                                      │
│  한글 번역                           │
│  나라는 무너졌으나 산하는 남아있고...  │
│                                      │
│  ▸ 집평 (접기/펼치기)                │  ← 기본 접힘
│  ▸ 주석 (접기/펼치기)                │  ← 기본 접힘
│  ▸ 심화자료 (접기/펼치기)            │  ← 기본 접힘
│                                      │
└──────────────────────────────────────┘
```

### 왜 이 방식이 좋은가

1. **모드 분기 불필요** — 짧은 시든 긴 시든 같은 구조
2. **동양화 느낌 살림** — 그림 위에 시가 올라가는 형의 욕심 충족
3. **긴 시 문제 해결** — 그림 영역을 벗어나면 종이 배경으로 자연스럽게 이어짐
4. **정보 과밀 해소** — 번역만 기본 노출, 나머지는 접힘
5. **모바일 대응 쉬움** — 1컬럼이라 반응형 자연스러움

## 4. 구체적인 CSS/HTML 구조

### 4-1. 모달 폭 복귀

```css
.modal {
  width: min(920px, 96vw);  /* 1400px → 920px */
}
```

동의: 세 문서 모두 900~980px 추천. 좁혀야 시의 세로 리듬이 살아남.

### 4-2. 그림 영역: 그라데이션 페이드

```css
.poem-hero {
  position: relative;
  background-size: cover;
  background-position: center;
  min-height: 300px;  /* 기본 높이 */
  padding: 40px 32px;
}

/* 하단 그라데이션: 그림 → 종이색 */
.poem-hero::after {
  content: '';
  position: absolute;
  bottom: 0; left: 0; right: 0;
  height: 120px;
  background: linear-gradient(transparent, #faf8f5);  /* 종이색 */
  pointer-events: none;
}
```

- 태훈의 "전략 B"를 발전시킨 것
- 그림의 하단이 자연스럽게 종이색으로 사라짐
- 시 텍스트는 absolute가 아니라 **일반 흐름(flow)**으로 배치
  - 현재 문제: `position: absolute` + `max-height: 85%` → 긴 시가 잘림
  - 해결: 일반 흐름이면 시가 길어도 영역이 자연스럽게 늘어남

### 4-3. 시 텍스트: absolute 대신 일반 흐름

```css
.poem-hero-text {
  position: relative;  /* absolute → relative */
  z-index: 1;          /* 그라데이션 위에 */
  color: #fff;
  text-shadow: 0 1px 8px rgba(0,0,0,0.5);  /* 가독성 */
  font-family: fangsong, 'LXGW WenKai Mono TC', serif;
  font-size: 20px;
  line-height: 2.0;
  text-align: center;
  padding: 60px 40px 80px;
}
```

핵심 변경: **absolute → relative**
- 짧은 시: 그림 안에 쏙 들어감 (min-height로 보장)
- 긴 시: 자연스럽게 아래로 늘어남 → 그라데이션 지나서 종이 배경으로 전환
- 더 이상 `max-height`로 잘리지 않음

### 4-4. 긴 시 자동 감지 → 배경 처리 조절

```javascript
// 시 행수에 따라 그림 높이 자동 조절
const lineCount = poemZh.split('\n').filter(l => l.trim()).length;

if (lineCount <= 8) {
  // 절구/율시: 그림이 시 전체를 감싸도록
  heroEl.style.minHeight = '400px';
} else if (lineCount <= 16) {
  // 배율/고시: 그림 높이 적당히
  heroEl.style.minHeight = '300px';
} else {
  // 장편 (백거이 등): 그림은 상단 배너 역할
  heroEl.style.minHeight = '250px';
  // 시 텍스트 색상을 전환: 그림 부분=흰색, 종이 부분=검정
}
```

이러면 Art/Study 모드 분기 없이도 긴 시가 자연스럽게 처리됨.

### 4-5. 학습 영역: 기본 접힘 (아코디언)

```
번역 (기본 펼침)
──────────────────

▸ 집평 보기          ← 클릭하면 펼침
▸ 주석 보기          ← 클릭하면 펼침
▸ 심화자료 보기      ← 클릭하면 펼침
  ├ 병음 | 평측 (2컬럼)
  ├ YouTube
  └ TTS
```

현재 문제: 모든 섹션이 다 펼쳐져 있어서 끝이 안 보임
해결: 번역만 기본 노출, 나머지는 접어두기
- 학술적 목적의 사용자만 필요할 때 펼치면 됨
- 이건 기존 시인모달의 작품 아코디언과 동일한 UX

## 5. 그림 관련 추가 아이디어

### 5-1. 그림이 없는 시 처리

미드저니 그림이 아직 없는 시도 많을 거야. 이때는:
- 시대별 기본 텍스처 배경 사용 (현재 랜덤 배경과 유사하지만 더 은은하게)
- 또는 종이 질감만 깔기
- 그림이 추가되면 자동으로 전환

### 5-2. 그림 투명도/블렌딩 프리셋

관리툴에서 시별로 조절:
- `강조 (0.9)`: 그림이 선명, 텍스트에 그림자 강하게 → 그림이 주인공인 시
- `조화 (0.6)`: 기본값, 그림과 텍스트 균형
- `은은 (0.3)`: 그림이 워터마크처럼, 텍스트 가독성 최우선

### 5-3. 세로쓰기 옵션 (장기)

형이 "동양화 위에 시를 써넣는 전통"을 언급했는데, CSS `writing-mode: vertical-rl`을 쓰면 세로쓰기가 가능해. 짧은 절구/율시에 특히 잘 어울림. 다만 이건 Phase 3 이후에 추가하는 게 좋을 것 같아.

## 7. 구현 단계 (Phase)

### Phase 1: 즉시 효과 (빠른 복구)
- 모달 폭 1400px → 920px
- 본문 외 섹션 (집평/주석/심화) 기본 접힘으로 변경
- `.modal-title` 다시 표시
- 예상 효과: "이상하다"는 느낌 70% 해소

### Phase 2: 그림 + 시 레이아웃 개편
- `.poem-bg-wrap` → `.poem-hero` 구조 변경
- 그라데이션 페이드 적용
- 시 텍스트 `absolute → relative` 전환
- 행수 기반 자동 높이 조절
- 관리툴 미리보기 동기화

### Phase 3: 완성도 + 디테일
- 그림 투명도 프리셋 (관리툴에서 선택)
- 그림 없는 시 기본 텍스처
- 세로쓰기 옵션 (선택적)
- 모바일 반응형 최종 점검
- QA: 긴 시 10편 + 짧은 시 10편 + 그림 있는/없는 각 5편

## 10. 결론

"완벽한 하나의 레이아웃"은 없다. 하지만 **두루마리 컨셉(상단 감상 → 하단 학습, 자연스러운 전환)**이 형이 원하는 방향과 가장 가깝고, 기술적으로도 가장 안전하다.

형이 포기하기 싫었던 "그림 위에 시" — 이건 살릴 수 있다. 다만 지금처럼 `absolute`로 그림 안에 가두는 게 아니라, **시가 주인공이고 그림이 배경으로 자연스럽게 깔리는** 방식으로 바꾸면 긴 시든 짧은 시든 다 소화할 수 있다.

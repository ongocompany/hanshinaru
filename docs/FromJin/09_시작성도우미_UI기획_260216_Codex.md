# 근체시 작성 도우미 UI 기획안 (화면/플로우 중심)

기준 문서:
- `docs/FromJin/06_시작성도우미제작계획to지훈.md`

목표:
- 규칙 학습과 창작 실습을 끊김 없이 연결
- 초보자도 시작 가능하고, 숙련자도 답답하지 않은 UX

---

## 1. 정보 구조(IA)

### 상단: 학습(읽기)
- 근체시 핵심 규칙 요약 카드
- “더 보기”로 상세 규칙 확장

### 중단: 예시(비교)
- 시체 탭: 오언절구/칠언절구/오언율시/칠언율시/배율
- 대표 작품 카드 + 하이라이트 토글
  - 압운 표시
  - 대우 표시
  - 평측 힌트

### 하단: 실습(쓰기)
- 7단계 입력 도우미 + 모드 토글
- 우측 고정 패널: 검사 요약/추천 후보/도움말

---

## 2. 핵심 화면 컴포넌트

## 2.1 Stepper(7단계)
- 상단 고정 진행바
- 각 단계 완료 시 체크 + 다음 단계 활성화
- “이전 단계 수정” 즉시 가능

## 2.2 모드 토글
- `자유 작성 + 일괄 검사`
- `실시간 가이드`
- 모드 전환 시 입력값 유지(절대 초기화 금지)

## 2.3 작성 영역
- 라인 단위 입력칸(4구/8구 자동)
- 현재 라인 상태칩:
  - 글자수
  - 압운 슬롯
  - 평측 상태

## 2.4 우측 보조 패널
- 탭:
  1. 규칙 요약
  2. 운서 검색
  3. 후보 추천
  4. 오류 목록
- 실시간 모드에서 가장 자주 쓰는 영역

---

## 3. 사용자 플로우

### 플로우 A (초보)
1. 형식 선택
2. 주제/소재 입력
3. 구상(한글) 작성
4. 운부/운자 확정
5. 실시간 모드로 한 글자씩 작성
6. 학인 실행
7. 감상 카드 + 검증 번역 확인

### 플로우 B (숙련)
1. 형식 선택
2. 바로 본문 작성
3. 일괄 검사
4. 오류 최소 수정
5. 저장/공유

---

## 4. 시각/인터랙션 규칙

### 4.1 신호등 색상 체계
- 빨강: 즉시 수정 필요
- 노랑: 선택 경고
- 초록: 통과
- 회색: 아직 미검사

### 4.2 오류 표시 방식
- 텍스트 전체 빨갛게 금지
- “문제 위치(글자 슬롯)”에만 배지 표시
- 하단 오류 리스트 클릭 시 해당 글자로 스크롤/포커스

### 4.3 후보 선택 UX
- 후보 클릭 = 임시 적용(ghost)
- “적용” 버튼 눌러야 확정
- 사용자 통제권 유지

---

## 5. Step별 UI 상세

### Step 1 형식
- 카드형 선택 + 예시 한 줄
- 고급옵션은 접기(평기/측기/입운)

### Step 2 주제·소재
- 입력칸 + 태그 입력
- AI 보조 버튼: “소재 재정리”

### Step 3 구상(한글)
- 라인별 메모 칸
- 율시는 련별 안내 라벨 고정

### Step 4 운목
- 운부 검색 + 분위기 설명
- 선택 운자 pin 고정

### Step 5 압운 배치
- 압운 슬롯만 먼저 채우는 전용 UI
- 잘못된 운부는 즉시 차단

### Step 6 본문 작성
- 모드A: 대형 텍스트 입력
- 모드B: 슬롯 입력 + 추천 패널 + 신호등

### Step 7 감상/리포트
- 작품 카드(가독성 중심)
- 접힘 리포트:
  - 형식 진단
  - 대우 진단
  - 검증 번역

---

## 6. 모바일/반응형

1. 데스크탑: 2열 (작성 + 보조패널)
2. 태블릿: 1.5열(보조패널 접힘)
3. 모바일: 단일열 + 하단 시트 패널

모바일 우선순위:
- 작성칸 포커스 유지
- 오류 이동 버튼
- 후보 패널 바텀시트

---

## 7. 저장/공유 UX

### v1 (프론트-only)
- 버튼:
  - 임시저장
  - 버전저장
  - JSON 내보내기
  - JSON 가져오기
  - 공유 링크 생성
  - 시화 PNG 다운로드

### v2 (선택)
- 로그인 후 클라우드 저장
- 공개/비공개 전환
- 작품 상세 퍼머링크

---

## 8. 접근성(A11y)

1. 신호등 색상만으로 의미 전달 금지
- 아이콘/텍스트 병행(예: `오류`, `경고`, `통과`)

2. 키보드 작성 지원
- 라인 이동 단축키
- 오류 간 점프 단축키

3. 스크린리더 대응
- 검사 결과는 aria-live 영역으로 전달

---

## 9. 내가 추가 제안하는 UI 아이디어

1. **학습모드 오버레이**
- 입력칸 위에 “이 줄은 역할이 무엇인지” 표시(제시/대우/결구)

2. **수정 시뮬레이터**
- 후보 클릭 시 예상 점수/경고 변화 미리보기

3. **감상 테마 전환**
- 학술 보기 / 작품 보기 스킨 분리

4. **작성 집중 모드**
- 불필요 패널 숨기고 본문과 최소 피드백만 노출

---

## 10. 우선 구현 체크리스트

1. 7단계 stepper 뼈대
2. 모드 전환 상태 유지
3. 엔진 오류 좌표 하이라이트
4. 운서 검색 패널
5. 학인 리포트 기본 화면
6. 저장/내보내기

# 시인 모달 및 관리툴 UI 개선 제안서

작성일: 2026-02-15
작성자: Gemini (태훈)
수신: JIN (형님)
목적: 미드저니 이미지와 텍스트(시, 해설)의 부조화를 해결하고 심미성과 가독성을 동시에 확보하는 UI 전략 수립.

## 1. 문제의 본질 분석

형님이 느끼는 "이상함"의 원인은 **정보의 성격이 다른 요소들의 충돌** 때문입니다.

1.  **시(Poem) + 그림(Image)**: **감상(Appreciation)**의 영역. 여백, 운율, 분위기가 중요합니다. (우뇌 자극)
2.  **주석, 집평, 번역**: **학습(Study)**의 영역. 논리적이고 빽빽하며 가독성이 중요합니다. (좌뇌 자극)

이 둘을 위아래로 단순히 쌓거나(Stack), 좁은 공간에 억지로 끼워 넣으려니 시각적 리듬이 깨지는 것입니다. 특히 **"미치게 긴 시(장한가, 비파행 등)"**가 레이아웃을 파괴하는 주범입니다.

## 2. 해결 방안 제안

### 전략 A: "감상"과 "학습"의 공간 분리 (2-Column Sticky Layout)

모달의 폭을 900px~1100px 정도로 넓히고, 화면을 좌우로 나눕니다.

-   **좌측 (감상존, 40~50%)**: **[이미지 + 시 원문]**
    -   이곳은 "예술적인 느낌"을 담당합니다.
    -   **핵심 기술**: `position: sticky`
    -   시가 짧으면: 이미지와 시가 중앙에 예쁘게 배치됩니다.
    -   시가 길면: **이미지는 화면에 고정(Sticky)되어 있고, 시 텍스트만 위로 스크롤**되면서 올라갑니다. (마치 영화 엔딩 크레딧이나 박물관 오디오 가이드처럼)
    -   배경: 미드저니 이미지를 은은하게 깔거나(Blur), 캔버스 질감을 줍니다.

-   **우측 (학습존, 50~60%)**: **[번역 + 주석 + 집평]**
    -   이곳은 "학습 웹페이지" 느낌을 유지합니다.
    -   흰색(또는 종이색) 배경에 깔끔한 검은 글씨로 가독성을 최우선으로 합니다.
    -   좌측의 시와 독립적으로 스크롤됩니다.

### 전략 B: 동양화 여백 효과 (Gradient Masking)

형님이 말씀하신 "그림이 바닥에 깔리고 그 위에 시가 올라가는" 방식을 구현하되, 가독성을 해치지 않는 방법입니다.

1.  **이미지 배치**: 미드저니 그림을 모달의 **상단 배경(Header Background)**으로 넓게 깝니다.
2.  **그라데이션 마스크**: 그림의 하단부로 갈수록 투명해지면서 본문 배경색(종이색)과 자연스럽게 섞이게 합니다. (`mask-image: linear-gradient(...)`)
3.  **시 배치**: 그림의 여백(또는 마스킹된 부분) 위에 시 원문을 세로쓰기(또는 명조체 가로쓰기)로 얹습니다.
4.  **긴 시 처리**: 시가 길어지면 그림 영역을 벗어나 자연스럽게 단색 배경 위로 글자가 이어지게 합니다. 이렇게 하면 그림 크기에 구애받지 않고 긴 시도 처리가 가능합니다.

## 3. 구체적인 레이아웃 아이디어 (스케치)

**[제안: 갤러리형 모달]**

```text
-------------------------------------------------------
|  [ 닫기 ]                                           |
|                                                     |
|   [ 좌측: 고정된 비주얼 영역 ]   |  [ 우측: 스크롤되는 정보 영역 ]  |
|                                |                      |
|   +------------------------+   |  ## 번역             |
|   |                        |   |  (번역문 텍스트...)    |
|   |   미드저니 이미지        |   |                      |
|   |   (Sticky 고정)         |   |  ## 집평             |
|   |                        |   |  (집평 텍스트...)      |
|   +------------------------+   |                      |
|                                |  ## 주석             |
|   (시 원문이 이미지 위로      |   1. 주석 내용...      |
|    오버레이되거나 아래에      |   2. 주석 내용...      |
|    위치하여 스크롤됨)         |                      |
|                                |                      |
-------------------------------------------------------
```

## 4. 벤치마킹 추천 사이트

형님이 원하시는 "담백하면서도 예술적인, 그러면서도 정보 전달이 잘 되는" 사이트들입니다.

1.  **Google Arts & Culture (Story 형태)**
    -   특징: 고해상도 명화를 배경에 깔고, 텍스트가 그 위를 부드럽게 흘러가거나 옆에 고정됩니다. 긴 호흡의 콘텐츠를 보여주는 최고의 레퍼런스입니다.
    -   참고 포인트: 스크롤에 따른 이미지 페이드인/아웃 처리.

2.  **Medium (Article Layout)**
    -   특징: 텍스트 가독성의 끝판왕입니다. 이미지를 본문 너비보다 넓게(Full-width) 쓰거나, 텍스트 옆에 배치하는 방식이 매우 세련되었습니다.
    -   참고 포인트: 여백의 활용과 타이포그래피.

3.  **국립고궁박물원 (대만/한국)**
    -   특징: 유물(이미지)과 설명(텍스트)을 배치하는 방식이 매우 정석적이고 학술적입니다.
    -   참고 포인트: 동양적인 색감과 여백 활용.

## 5. 결론 및 다음 단계

-   **추천**: **전략 A (좌우 분할 + Sticky 이미지)**가 가장 현실적이고 구현 가능성이 높습니다. 긴 시(백거이) 문제도 해결되고, 그림도 살릴 수 있습니다.
-   **액션 아이템**:
    1.  `styles.css`에서 모달 레이아웃을 `display: grid` 또는 `flex`로 변경하여 좌우 분할 구조를 잡는다.
    2.  좌측 패널에 `position: sticky`를 적용하여 이미지가 따라오게 만든다.
    3.  우측 패널에 해설 텍스트를 몰아넣는다.

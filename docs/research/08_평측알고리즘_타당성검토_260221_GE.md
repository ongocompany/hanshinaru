## 1. hanshi.kr과 우리 프로젝트(작성 도우미)의 비교

### hanshi.kr의 주요 검사 기능
- 각 글자의 평(○), 측(●) 식별
- **고급 병폐 검출**: 하삼평(下三平), 하삼측(下三仄), 봉요(蜂腰), 학슬(鶴膝) 등

### 우리 "근체시 작성 도우미"의 기획 체급
민철이 형이 정리한 `01_설계검토보고서`의 **[C-1. 병(病) 탐지 범위 불완전]** 항목과 `03_개발로드맵`의 **[Phase 2: 엔진 코어]** 2-4, 2-5 태스크를 보면, 우리는 이미 hanshi.kr을 능가하는 검사 로직을 기획에 포함해 두었습니다.

우리의 검출 타겟:
1.  **기본 격률 위반**: 정격 룰 위반
2.  **형태적 병폐**: 고평(孤平), 삼평미(三平尾 = 하삼평), 삼측미(三仄尾 = 하삼측), **봉요(蜂腰), 학슬(鶴膝)**
3.  **운율적 병폐**: 대운(大韻), 소운(小韻)
4.  **고급 룰 (요구 拗救)**: 단순히 틀렸다고(빨간불) 처리하지 않고, 시인이 의도적으로 룰을 어기고 다른 글자로 보상하는 '요체' 패턴(10~15종)까지 인식하여 허용(노란불)하는 로직 포함. *(이 부분은 일반적인 기계적 평측사전 검사기를 뛰어넘는 수준입니다.)*

## 2. 고급 로직의 JS 알고리즘 구현 가능성 

웹 브라우저에서 돌아가는 순수 자바스크립트(Vanilla JS) 만으로도 이 모든 것을 매우 빠르고 정확하게 처리할 수 있습니다. 한시의 평측 배열은 일종의 **정규 표현식(Regular Expression)이나 1차원 배열 패턴 매칭** 문제와 정확히 일치하기 때문입니다.

### 구현 예제 (논리 구조)

#### 1) 삼평미(하삼평) 판별 알고리즘
- **로직**: 각 句(구)의 글자 배열에서 마지막 3글자의 평측 데이터 배열을 자릅니다.
- **JS 수도코드**: 
  ```javascript
  const lineTones = ["평", "측", "평", "평", "평"]; // 5언시 예시
  const lastThree = lineTones.slice(-3); // ["평", "평", "평"]
  if (lastThree.every(tone => tone === "평")) {
      return "경고: 삼평미(하삼평) 발생!";
  }
  ```

#### 2) 봉요(蜂腰) 판별 알고리즘
- **로직**: 5언시 기준, 2번째 글자와 4번째 글자의 평측이 동일한지 검사합니다. (예: `○ ● ○ ● ○`)
- **JS 수도코드**:
  ```javascript
  const lineTones = ["평", "측", "평", "측", "평"]; 
  if (lineTones[1] === lineTones[3]) { // 인덱스 1(두번째), 3(네번째) 비교
      return "경고: 봉요(蜂腰) 발생! 2자, 4자의 평측이 같습니다.";
  }
  ```

#### 3) 운율 스코프 검토 (대운, 소운)
- 한 구 안에서, 또는 련 단위에서 한자의 `rhyme_key(운부)` 속성을 비교하여 중복이나 충돌을 찾아냅니다. 이는 우리가 구축할 `char_map v2` DB에 평측과 운부 데이터가 모두 들어있기 때문에 단순 객체 키 비교만으로 순식간에 끝납니다.

## 3. 태훈이의 추가 제언 (우리 시스템의 초격차 우위)

hanshi.kr의 평측 사전과 비교해 우리가 준비 중인 방식이 압도적인 우위를 가지는 이유는 **"AI와의 결합 방식"**과 **"UX(사용자 경험)"**에 있습니다.

1.  **다음자(多音字)와 입성(入聲)의 정밀 처리**: 
    - 일반 사전은 그냥 평/측을 기계적으로 띄우고 "문맥에 따라 다름(❉)"으로 퉁기곤 합니다. 
    - 우리 기획(설계서 A-1, A-2)은 이를 배열 구조로 세분화하고, AI 프롬프트를 통해 "이 문맥에서는 측성으로 읽는 것이 맞습니다"라고 코칭해주는 단계(Phase 4)까지 계획되어 있습니다.
2.  **검사가 아닌 '창작 지원'**:
    - 일반 시스템은 "틀렸어!"라고 에러만 띄우지만(사후 검사), 
    - 우리는 원고지 UI(설계서 H항목)에서 3색 신호등을 실시간 파이프라인으로 돌리면서, 애초에 글자를 고를 때(IME 창) 평측에 맞는 후보 시어들을 확률 높게 추천해줍니다. (검사 전 오류 방지)

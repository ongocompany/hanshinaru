# 시인 모달 및 시관리툴 UI 개선 제안서

원본 문서:
- `docs/FromJin/UI_developing_brainstorming.md`

## 1) 결론 먼저

핵심 문제는 "정보 과밀 + 표현 목적(예술)과 기능 목적(학습)이 한 레이아웃에 동시에 붙어 있는 것"입니다.  
해결은 **하나의 완벽한 레이아웃을 찾는 것**이 아니라, **시 길이/용도에 따라 2가지 보기 모드로 분기**하는 것이 가장 안전합니다.

- `Art 모드`: 짧은 시(절구/율시) 중심, 그림+여백+시 본문 감상
- `Study 모드`: 긴 시(고시/악부/백거이 장편) 중심, 텍스트 가독성과 자료 탐색

## 2) 현재 코드 기준 진단(문제 근거)

1. 모달 폭/의도 불일치
- 주석은 "폭 900px"인데 실제는 `width: min(1400px, 96vw)`입니다.
- 근거: `styles.css:566`, `styles.css:580`
- 결과: 가로가 넓어지며 세로형 본문/가로형 보조문서가 같은 시야에 충돌.

2. 모달 헤더 정보 약화
- 모달 제목이 강제로 숨김 처리되어 컨텍스트가 약함.
- 근거: `styles.css:598`

3. 본문/집평/주석/상세/심화가 단일 수직 흐름에 과적재
- 한 섹션 내부에 자료군이 연속 적층되어 "감상"과 "참조"의 리듬이 없음.
- 근거: `app.js:1245`, `app.js:1259`, `app.js:1264`, `app.js:1269`, `app.js:1280`

4. 그림 위 텍스트 오버레이가 긴 시에서 역효과
- `max-height: 85%` + `overflow: auto`라 긴 시는 "그림 위 작은 스크롤 박스"가 됨.
- 근거: `styles.css:1378`

5. 주석 레이아웃이 고정폭이라 시각적 압박
- `grid-template-columns: 48px 120px 1fr`
- 근거: `styles.css:840`
- 긴 텍스트/좁은 화면에서 줄맞춤이 무거워 보임.

## 3) 개선 방향(실전형)

### A. 레이아웃 전략: 모드 분리

1. Art 모드 (기본)
- 적용 대상: 본문 길이 짧은 시
- 구성:
  - 상단: 제목/시인
  - 본문: 배경 그림 + 시 텍스트 오버레이
  - 하단: "집평/주석 보기"는 접힘 패널
- 포인트: 한 화면에 "한 작품의 기분"을 전달

2. Study 모드 (자동 전환 + 수동 토글)
- 적용 대상: 긴 시(예: 24행 이상, 또는 문자수 기준)
- 구성:
  - 본문은 일반 배경(white/ivory)에서 가독성 우선
  - 그림은 우측 사이드 또는 상단 배너(낮은 대비)
  - 집평/주석/번역은 탭으로 분리
- 포인트: 탐독/검색/비교를 빠르게

### B. 폭/간격 정리

1. 모달 폭 900~980px 우선 복귀
- 현재 1400px는 콘텐츠 밀도가 너무 높음
- 제안: 데스크탑 `920px`, 대형화면에서만 `max 1040px`

2. 섹션 계층 단순화
- 기본 노출: 시 본문 + (짧은 메타)
- 2차 자료(집평/주석/번역/심화): 탭 또는 접힘으로 이동

### C. "사각형 그림 어색함" 해결

1. 하드 사각 박스 대신 마스크/그라디언트 블렌딩
- 하단 우측 앵커 이미지 + 가장자리 페이드
- 배경 위 텍스트 박스는 완전 불투명 박스 대신 반투명/무배경 옵션

2. 텍스트와 배경 분리 레벨 제공
- `강조`: 배경 진하게, 텍스트 상자 옅게
- `중간`
- `가독`: 배경 옅게(거의 워터마크)

### D. 긴 시 처리 규칙(필수)

1. 자동 규칙
- 행 수/문자수 임계치 넘으면 Study 모드 기본

2. 사용자 토글
- "예술 보기 / 학습 보기" 즉시 전환 버튼 제공

3. 본문 내 탐색
- 긴 시는 연 단위 앵커(또는 1~N 블록 분할) 지원

## 4) 단계별 구현 제안 (리스크 낮은 순)

### Phase 1 (빠른 복구, 1~2일)
- 모달 폭 조정, 헤더/간격 정리
- 본문 외 자료 기본 접힘
- 목표: "지금 이상한 느낌" 즉시 해소

### Phase 2 (핵심, 2~4일)
- Art/Study 모드 분기 로직 추가
- 긴 시 자동 Study 모드 전환
- 집평/주석/번역 탭화

### Phase 3 (완성도, 2~3일)
- 배경 블렌딩/마스크 스타일
- 텍스트-배경 대비 프리셋
- 시각 QA (긴 시 10개, 짧은 시 10개)

## 5) 너의 고민별 직접 답변

1. "900px로 되돌리면 나아질까?"
- 네, 바로 체감됩니다. 지금은 폭이 너무 넓어 레이아웃 충돌이 커진 상태입니다.

2. "그림+시 한 화면 최선?"
- 짧은 시는 한 화면 가능. 긴 시는 분리해야 품질이 올라갑니다.

3. "우하단 그림 크롭 어색"
- 단순 사각 배치 대신 페이드 마스크 + 워터마크 톤이 정답에 가깝습니다.

4. "동양화처럼 여백+시"
- 맞는 방향입니다. 다만 긴 시까지 동일 룰로 밀어붙이면 실패 확률이 큽니다.

5. "백거이 같은 장편"
- Study 모드 강제 전환 + 탭/앵커 탐색이 사실상 필수입니다.

## 6) 벤치마크 사이트 추천 (2026-02-15 확인)

1. Chinese Text Project (고전 텍스트 탐색 UX)
- https://ctext.org/
- 이유: 본문/주석/검색/교차참조 구조가 학술형 읽기에 강함.

2. Wikisource 중국어판 唐詩三百首 (목차/문헌형 구조)
- https://zh.wikisource.org/wiki/%E5%94%90%E8%A9%A9%E4%B8%89%E7%99%BE%E9%A6%96
- 이유: 길고 많은 텍스트를 안정적으로 네비게이션하는 방식 참고용.

3. Poetry Foundation (현대 시 감상 페이지 구조)
- https://www.poetryfoundation.org/
- 이유: 시 본문 집중 + 보조정보 분리의 균형이 좋음.

4. Academy of American Poets / Poets.org
- https://poets.org/poem-a-day
- 이유: 감상형 뷰와 아카이브형 뷰를 분리해 제공.

5. Poetry in Translation (고전 번역 큐레이션)
- https://www.poetryintranslation.com/
- 이유: 고전시를 읽기 흐름 중심으로 배치하는 참고 사례.

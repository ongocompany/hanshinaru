## 4) 검색 파라미터(최종)
사용 언어 범위:
- 한국어(ko)
- 중국어 번체(zh-Hant)
- 중국어 간체(zh-Hans)
- 일본어(ja)

검색 규칙:
- 이름 자체 중심 검색
- 간체/번체/이체 표기 모두 시도
- `생몰`, `생졸`, `출생`, `사망`, `born`, `died` 등 생몰연도 중심 키워드는 제외
- 대신 아래 맥락 키워드를 우선 사용:
  - `活动时期`, `时代`, `大历十才子`, `开元`, `元和`, `天宝`, `晚唐`, `中唐`, `盛唐`, `初唐`
  - `詩人`, `人物`, `傳`, `年號`

검색 템플릿:
- `"<이름>" + "活动时期" + "唐"`
- `"<이름>" + "时代" + "唐诗"`
- `"<이름>" + "开元|天宝|元和|大历"`
- `"<이름>" + "中唐|晚唐|盛唐|初唐"`
- `"<이름>" + "詩人" + "生平"` (생몰 키워드 미포함)

## 5) 검색 히스토리(요약)
### 5-1. 1차 조사
- 목적: `birth == null` 작가의 시대 분류 가능성 확인
- 방식: 이름 단위 웹 검색 + 시대명/연호/관련 인물 단서 수집
- 결과: 일부 작가는 시대 직접 표기 문헌을 확보했고, 일부는 단서가 약해 보류

### 5-2. 사용자 조건 수정 반영
- 변경사항:
  - 검색 범위를 간체까지 확대
  - 생몰년 중심 키워드 사용 금지
- 상태:
  - 수정된 파라미터 기준으로 후속 검증 진행 예정

### 5-3. 쿼리 로그 형식(재사용용)

## 7) 후속 작업 가이드(다른 AI용)
1. 위 `추가 조사 필요` 13명을 우선 처리한다.
2. 동일 이름의 동명이인 충돌을 먼저 제거한다(예: 唐代 詩人 여부 확인).
3. 생몰연도 숫자 자체보다, 아래 우선순위로 근거를 모은다.
   - 문헌의 시대 직접 표기(초당/성당/중당/만당)
   - 연호/사건/동시대 인물 교유 정보
   - 작품의 편찬 시기/과거 급제 연도
4. 최종 결과를 `era` 후보값으로 구조화한다.
   - `eraCandidate`: `early|high|mid|late|range|unknown`
   - `evidenceType`: `direct_era|regnal_year|related_person|weak`
   - `confidence`: `high|medium|low`
5. 최종 반영 시 `db_author.with_ko.json` 직접 필드 추가 여부는 별도 합의 후 진행한다.

## 8) 재현용 로컬 명령
`birth == null` 작가 추출:

```bash
node - <<'NODE'
const fs=require('fs');
const db=JSON.parse(fs.readFileSync('public/index/db_author.with_ko.json','utf8'));
const authors=Object.values(db.authors||{});
const noBirth=authors
  .filter(a=>a?.life?.birth==null)
  .map(a=>({id:a.titleId,zh:a?.name?.zh||'',ko:a?.name?.ko||''}))
  .sort((a,b)=>a.zh.localeCompare(b.zh,'zh-Hant'));
console.log(JSON.stringify({count:noBirth.length,noBirth},null,2));
NODE
```

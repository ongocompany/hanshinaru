# 목표

한시나루의 목표를 "한국에서 운영하고 서비스하는 한시 관련 사이트 중 가장 많은 한시 제공"으로 둔다.

이번 문서는 당대/전당시와 섞이지 않는 `CN non-Tang` 수집 대상표다. JDS/전당시 파이프라인은 수집·정규화·저장 방식만 참조하고, 당나라 시 원문·시인·권차는 이번 수집 대상에서 제외한다.

# 기본 원칙

1. 당대는 제외한다. `初唐`, `盛唐`, `中唐`, `晩唐`, 미분류 `唐`은 이 문서의 대상이 아니다.
2. 위키문헌을 1차 기본 소스로 삼는다.
3. 제공 원문은 간체자가 아니라 번체 기반 한국식 한자를 기본값으로 한다.
4. 대표작 10수 공개가 아니라, 가능한 많은 시인·작품을 후보화한다.
5. DB 반영 전에는 반드시 `raw source`, `normalized record`, `review queue`, `dry-run counts`를 남긴다.

# 수집 순서

1. `宋`, `明`의 시대 category를 먼저 긁어 대량 후보 pool을 만든다.
2. `兩漢`, `魏晉南北朝`는 선집·문집 기반으로 정확도를 높인다.
3. `元`, `淸`은 category 전량 후보화 후 부족한 작가를 시집으로 보강한다.
4. 모든 원문은 저장 전 `normalizeChineseForHanshinaru()`로 한국식 한자 정규화를 통과시킨다.
5. 번역은 대량 수집과 분리한다. 먼저 원문/메타를 많이 모으고, 공개 우선순위부터 번역한다.

# 다음 작업

1. 이 문서를 기준으로 `cn_non_tang_collection_targets.v1.json`을 만든다.
2. 위키문헌 category API에서 `宋詩`, `元詩`, `明詩`, `清詩` page list를 raw로 저장한다.
3. page title에서 작가명이 괄호로 들어간 항목을 우선 파싱한다.
4. 작가 분류가 큰 `蘇軾`, `陸游`, `陶淵明`, `高啓`, `龔自珍`부터 author page 수집기를 붙인다.
5. DB 적재는 수집/정규화/검토 큐가 생긴 뒤 다시 결정한다.

# AI 저작툴 (AI Writer) 설계 문서

## 1. 개요

한시나루 웹사이트의 콘텐츠를 효율적으로 작성하기 위한 AI 보조 저작 도구.
사용자가 주제와 스타일을 설정하면 AI가 초안을 생성하고, 검토/편집 후 Supabase DB에 저장한다.

### 핵심 플로우

```
주제/설정 입력 → AI 초안 생성 → 검토/편집 → DB 저장
```

### 배치

- 경로: `tools/ai-writer/index.html`
- 형태: 단일 페이지 독립 도구 (Vanilla HTML/CSS/JS)
- 향후: admin 페이지 정비 후 통합 예정
x`
## 2. 콘텐츠 유형

| 유형 | 대상 |
|------|------|
| 시 해설 | 개별 시에 대한 해설, 감상, 번역 |
| 시인 소개 | 시인 약력, 작품 세계 |
| 역사 카드 | 시대별 역사 사건 (당나라 + 확장 예정) |
| 아티클 | 문학사, 역사, 입문 가이드 등 장문 콘텐츠 |

## 3. AI 모델 및 API

### 모델

- 기본: Qwen 3.6 Plus (한시/한문/중국사 특화)
- OpenRouter 경유 사용 (현재 무료 프리뷰)

### 멀티 프로바이더 지원

프로바이더 교체가 자유로운 구조. 내부적으로 OpenAI chat completions 형식으로 통일하고, base URL과 헤더만 프로바이더별로 분기한다.

| 프로바이더 | Base URL | 용도 |
|-----------|----------|------|
| OpenRouter | `https://openrouter.ai/api/v1` | Qwen 3.6+, Claude 등 |
| Alibaba DashScope | `https://dashscope.aliyuncs.com/compatible-mode/v1` | Qwen 전용 |
| OpenAI | `https://api.openai.com/v1` | GPT 시리즈 |
| Google AI | `https://generativelanguage.googleapis.com/v1beta/openai` | Gemini |
| 커스텀 URL | 사용자 입력 | Ollama 로컬 등 |

### API 설정 UI

설정 패널(톱니바퀴 아이콘)에서 관리:
- 프로바이더 선택 (드롭다운)
- API Key 입력
- 모델명 선택 (프로바이더별 프리셋 + 직접 입력)
- Base URL (프로바이더별 자동, 커스텀 시 직접 입력)
- 연결 테스트 버튼
- localStorage에 저장

## 4. 화면 구조

### 레이아웃: 좌우 2패널

```
┌─ 상단 바 ──────────────────────────────────────────┐
│ 한시나루 AI 저작툴               [불러오기] [초기화] │
├─ 좌측 입력 패널 (320px) ─┬─ 우측 에디터 패널 ───────┤
│                          │                          │
│ 콘텐츠 유형 (칩 선택)     │ 모드 탭: 시해설|시인|     │
│  + 커스텀 추가 가능       │         역사|아티클      │
│                          │                          │
│ 주제 입력                │ [에디터 영역]              │
│                          │  섹션별 구분 표시          │
│ 문체 프리셋 (드롭다운)    │  플로팅 툴바 (선택 시)    │
│  + 직접 입력 영역         │  슬래시 커맨드 (빈 줄)    │
│                          │                          │
│ 분량 선택                │                          │
│ 글 구조 선택             │                          │
│ 추가 지시사항            │                          │
│                          │                          │
│ [초안 생성]              │                          │
├──────────────────────────┴──────────────────────────┤
│ 하단 저장 바: section/slug 설정   [Draft] [Published]│
└─────────────────────────────────────────────────────┘
```

### 좌측 입력 패널

#### 콘텐츠 유형 칩
- 기본 제공: 시 해설, 문학사, 역사, 작가 소개, 입문 가이드
- [+ 추가] 버튼으로 커스텀 유형 생성 가능
- 커스텀 칩은 x 로 삭제 가능
- localStorage에 저장되어 세션 간 유지

#### 문체 프리셋
- 기본 프리셋: 대중 교양, 학술적, 에세이풍, 저널리즘, 쉬운 설명
- 드롭다운 하단에 "직접 입력" 자유 텍스트 영역
- 커스텀 프리셋 추가/저장 가능

#### 분량
- 짧게 (~1000자) / 보통 (~3000자) / 길게 (~5000자)

#### 글 구조
- 모드별 기본 구조 프리셋 제공 (예: 시 해설 -> 서론->원문/번역->해설->감상)
- 커스텀 구조 입력 가능

#### 추가 지시사항
- 자유 텍스트 입력 (선택)

### 우측 에디터 패널

4개 모드를 상단 탭으로 전환.

#### 모드 1: 시 해설 (poems 테이블 정형)

고정 섹션:
- 원문 (body_zh)
- 한국어 번역 (translation_ko)
- 해설 (commentary_ko)
- 주석 (notes)
- 핀인 (pinyin)

각 필드별 개별 AI 재생성 버튼 제공.

#### 모드 2: 시인 소개 (poets 테이블 정형)

고정 섹션:
- 기본 정보 (name_zh/ko, birth/death_year, birthplace)
- 약력 (bio_ko)
- 시대 구분 (era_period)
- 관계 (relations)

#### 모드 3: 역사 카드 (history_cards 테이블 정형)

고정 섹션:
- 연도 (year)
- 사건명 (name_ko/zh)
- 요약 (summary)
- 상세 설명 (detail)
- 태그/주석 (tags/annotations)

#### 모드 4: 아티클 (articles 테이블 자유)

자유 섹션 구조:
- 섹션 추가/삭제/순서변경 가능
- 각 섹션에 제목(편집 가능) + 본문
- 섹션별 AI 재생성, 위/아래 이동, 삭제 버튼
- 저장 시 모든 섹션을 하나의 body HTML로 합침
- section 필드 선택: chinese-poetry, korean-poetry, hanja, writing-helper

## 5. AI 호출 UX

### 초안 생성

좌측 패널에서 설정 완료 후 "초안 생성" 버튼 클릭.
스트리밍 응답으로 에디터에 실시간 표시.

### 편집 중 AI 호출 (2가지 방식)

#### 플로팅 툴바 (텍스트 선택 시)
1. 텍스트를 드래그해서 선택
2. 선택 영역 위에 플로팅 툴바 표시 (B, I, U | AI)
3. AI 버튼 클릭 -> 말풍선 팝업
4. 지시 입력 (예: "좀 더 구체적인 예시를 들어서 풀어써줘")
5. "적용" 클릭 -> 선택 영역만 AI가 재작성

#### 슬래시 커맨드 (빈 줄에서)
1. 빈 줄에서 / 입력
2. 명령 팔레트 표시:
   - 이어서 작성
   - 다시 쓰기
   - 분량 조절
   - 톤 변경
   - (자유 입력도 가능)
3. 선택 또는 입력 -> 해당 위치에 AI가 콘텐츠 생성/수정

## 6. 프롬프트 시스템

### 구조

```
System Prompt = 역할 정의 + 문체 기본 규칙 + 문체 프리셋 + 출력 구조 + 분량 지시 + 출력 포맷
User Prompt = 주제 + 추가 지시사항
```

### 문체 기본 규칙 (모든 프리셋에 공통 적용)

시스템 프롬프트에 기본 레이어로 포함:
- 평서문 사용 -- 경어체(~합니다/~입니다) 금지
- 위키체 지양 -- "~이다", "~것으로 알려져 있다" 식의 건조한 나열 금지
- 서브컬쳐 투 지양 -- 과도한 수사, 감탄조 금지
- AI 특유의 패턴 금지 -- "물론~", "흥미롭게도~", "~라고 할 수 있다" 같은 상투적 표현 차단

### 출력 포맷

AI 응답에 섹션 구분자를 넣게 지시:
```
===SECTION:서론===
(내용)
===SECTION:원문과 번역===
(내용)
```
에디터에서 이 구분자를 파싱하여 섹션별로 분리 표시.

### 시스템 프롬프트 커스터마이징

고급 설정에서 시스템 프롬프트 직접 수정 가능 (기본값 제공).

## 7. DB 저장

### 저장 대상 자동 분기

| 에디터 모드 | 저장 테이블 | 키 |
|------------|-----------|-----|
| 시 해설 | poems | poem_no_str |
| 시인 소개 | poets | id |
| 역사 카드 | history_cards | id |
| 아티클 | articles | slug |

### 저장 상태

- Draft / Published 선택 버튼
- articles: status 필드로 관리
- poems/poets/history_cards: 별도 status 필드가 없으면 바로 저장

### 기존 데이터 수정

- "불러오기" 기능으로 기존 레코드를 에디터에 로드하여 수정 가능
- UPDATE 쿼리로 저장

## 8. 기술 스택

| 항목 | 기술 |
|------|------|
| UI | Vanilla HTML/CSS/JS (기존 tools/ 패턴) |
| 에디터 | contenteditable div + 커스텀 섹션 관리 |
| AI API | OpenAI 호환 chat completions (fetch, SSE 스트리밍) |
| DB | Supabase REST API (기존 클라이언트 재사용) |
| 저장소 | localStorage (API 설정, 커스텀 칩, 프리셋) |

## 9. 향후 확장

- admin 페이지 통합 (admin 정비 완료 후)
- 시대 확장 (당나라 이전/이후)
- 이미지 생성 연동 (기존 midjourney_prompt_generator 활용)
- TTS 연동 (tts-studio)

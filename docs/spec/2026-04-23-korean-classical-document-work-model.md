# 목적

- 한국 고전 문헌, 시집, 문집을 가능한 한 많이 디지털 보존한다.
- 서비스에서는 작품을 보기 쉽게 분리하되, 연구·검수·재가공을 위해 문헌 원문 전체를 함께 보존한다.

# 핵심 원칙

1. `문헌 층`과 `작품 층`을 분리한다.
2. 문헌 원문은 가능한 한 `권차 -> section -> 항목` 구조로 그대로 보존한다.
3. 시·가·부·기문 등 작품성 텍스트는 문헌 층에서 파생된 `작품 층` 자산으로 분리한다.
4. 작품 층은 반드시 자신이 나온 문헌 위치를 가리켜야 한다.
5. 이미지/OCR는 문헌 원문이 없거나 손상된 부분을 보정할 때만 쓴다.

# 층별 역할

## 문헌 층

- 대상: `동경잡기(東京雜記)`, `東文選`, `新增東國輿地勝覽`, 개인 문집, 시집
- 보존 단위:
  - 문헌명
  - 권차
  - section
  - 항목 원문
  - 설명문, 기문, 주석, 산문, 부록
- 용도:
  - 연구자료 보존
  - 출처 검증
  - 작품 재분리
  - OCR/이본 대조 기준본

## 작품 층

- 대상: 문헌 층에서 분리된 시, 가, 부, 기문 등 작품 단위 텍스트
- 보존 단위:
  - 제목
  - 지은이
  - 본문
  - 형식
  - 이칭
  - 원문 출처 위치
- 용도:
  - 한시나루 `시 보기`
  - 번역
  - 주석
  - 작가별/작품별 탐색

# 연결 구조

- 작품 층은 아래 정보를 반드시 가진다.
  - 문헌명
  - 권차
  - section
  - 항목 식별자
- 즉 작품은 독립 자산이지만, 항상 `어느 문헌의 어디서 왔는지`를 잃지 않는다.

# 수집 흐름

1. 문헌 raw 확보
2. 문헌 층 보존
3. 시문/작품 후보 분리
4. 작품 층 생성
5. 중복/이칭 정리
6. 번역/주석/서비스 반영

# 현재 적용 판단

- `동경잡기(東京雜記)`처럼 문집형 문헌은 반드시 이중 모델로 다루는 편이 맞다.
- 예를 들어 `栢栗寺` 아래 `全思敬西樓記`는 문헌 층의 설명문으로 보존하고,
  그 아래 `鄭知常詩`, `朴孝修詩`는 작품 층으로 분리해 서비스에 쓴다.
- 이 방식이면 `시 보기` 메뉴에서는 작품만 보여주되, 연구자료로서의 문헌 원문 가치도 함께 유지할 수 있다.

# 운영 결론

- 앞으로 목표는 `한시 작품 수집`보다 넓은 `한국 고전 문헌·작품 디지털화`다.
- 따라서 모든 수집 파이프라인은
  - 먼저 문헌을 보존하고
  - 그다음 작품을 파생시키는
  방향으로 설계한다.
